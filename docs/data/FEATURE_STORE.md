# Purpose

The Feature Store provides a single source of truth for all calculated features used
by ATHENA.

Instead of recalculating indicators in multiple services, features are:

- Calculated once
- Validated
- Versioned
- Stored
- Shared

This improves:

- Performance
- Consistency
- Explainability
- Reproducibility
- AI Training

---

# Design Principles

1. Calculate once, use many times.
2. Features are immutable after creation.
3. Every feature has a version.
4. Every feature has an owner.
5. Every feature has metadata.
6. Historical values are preserved.
7. Features are reproducible.

# Feature Metadata

Every feature contains:

| Field | Type | Description |
|------|------|-------------|
| feature_id | UUID | Unique identifier |
| feature_name | VARCHAR | Feature name |
| category | ENUM | Feature category |
| value | DECIMAL | Calculated value |
| symbol | VARCHAR | Stock symbol |
| timeframe | ENUM | Time interval |
| calculation_time | TIMESTAMP | Generated at |
| version | INTEGER | Feature version |
| source | VARCHAR | Data source |
| owner_service | VARCHAR | Service owner |
